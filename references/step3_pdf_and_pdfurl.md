# Step3：PDF 与 pdf_url

## PDF 内容

每个 `需修改` 素材生成一页详情 PDF，至少包含：

- 发布时间
- GMV
- 消耗
- 播放
- ROI
- CTR / CVR
- 3s / 5s / 10s 完播
- 原视频 OSS
- 对标视频 OSS
- 数据定位问题
- 具体修改建议
- 脚本建议1
- 脚本建议2

## PDF 文件名

- 直接用 `素材ID.pdf`

## pdf_url 回填

当用户给到：

- 文件名
- URL

时：

- 按 `素材ID.pdf -> pdf_url` 匹配
- 回填到：
  - `需修改` 表
  - 最终整表里的对应行

## 最终大表

最终大表除了总明细基础列，还要带：

- 对标_素材ID
- 对标视频oss
- 对标2_素材ID
- 对标2视频oss
- 数据定位问题
- 具体修改建议
- gemini优化后完整脚本建议
- gemini优化后完整脚本建议2（微润色版）
- pdf_url
