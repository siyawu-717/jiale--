# Step1：数据整理与看板

## 输入

- 两份账号近30天效果数据表
- 信息维护表
- 视频ID-OSS映射表

## 统一口径

- 先按素材名称识别 `商品`
  - 回味粉：命中 `回味粉 / 回头粉`
  - 小面酱：命中 `小面酱 / 重庆小面 / 小面`（不含「小面筋」）；排在干锅、水煮之前，避免「水煮小面」等被归为水煮酱
  - 干锅酱：命中 `干锅酱 / 干锅`
  - 水煮酱：命中 `水煮酱 / 水煮 / 水煮麻辣`
- 用信息维护表按 `素材ID -> 账号` 覆盖账号归属
- `oss_path` 按 `素材ID -> OSS路径` 回填

## 三分类阈值

- 回味粉：
  - GMV >= 500：可复制
  - GMV = 0：直接放弃
  - 其余：需修改
- 干锅酱：
  - GMV >= 100：可复制
  - GMV = 0：直接放弃
  - 其余：需修改
- 水煮酱：
  - GMV >= 100：可复制
  - GMV = 0：直接放弃
  - 其余：需修改
- 小面酱（与干锅酱、水煮酱同一套金额阈值）：
  - GMV >= 100：可复制
  - GMV = 0：直接放弃
  - 其余：需修改

## 需要出的 sheet

### 1. 总体明细表

范围（`scripts/run_jiale_0412_pipeline.py`）：

- **创建时间**：默认与主表「需修改」主窗口一致（当前为 **2026-04-12～2026-05-12**，标签 **0412–0512**）。
- 可通过 `--dashboard-create-start` / `--dashboard-create-end`（旧名 `--global-create-start/end`）单独覆盖，用于例如「只看某段上新日」而主窗仍为 0412–0512。
- 行内 **消耗 / GMV** 等为千川导出在该复盘消耗窗口下的素材级指标。

至少包含：

- 商品
- 账号
- 素材ID
- 素材名称
- 素材创建时间
- oss_path
- 消耗
- GMV
- ROI
- 播放
- 千次播放GMV
- CTR
- CVR
- 3s完播率
- 5s完播率
- 10s完播率
- 三类结果

### 2. 看板数据表

需要同时放这三块：

- **月消耗看板**：**消耗时间** = 主窗口 **0412–0512**（与千川导出/复盘一致）；**创建时间** = 不筛（导出内全部素材）。
- **月创建看板**：**消耗时间**同上；**创建时间** = 与「总体明细」相同的创建窗（默认 **0412–0512**）。
- **每周新创建看板**：在「月创建」的创建起止内，按 **自然周（周一～周日）** 切创建时间；**消耗时间**文案仍与上两块一致；最后一周若未满 7 天需在结论中保守解读。

展示要求：

- 每块第一行写清楚消耗时间和创建时间
- 商品列合并单元格
- 每条账号行要带：
  - 可复制：数量（占比）
  - 需修改：数量（占比）
  - 直接放弃：数量（占比）
- 右侧需要有 `简单数据结论`

结论里要写：

- 奥创和轩辕谁整体更好
- 条数、总GMV、平均GMV
- 是点击端、转化端，还是开头留人差异更大
- 是否因为某一条可复制爆款明显拉高了平均GMV

### 3. 历史跑量爆款

逻辑：

- 不先筛创建时间
- 把历史上能达到 `可复制` 阈值的都单独拿出来

要求：

- 保留 `asr`

### 4. 可复制 / 需修改 / 直接放弃

- 保留原三分类sheet
- 其中：
  - `可复制` 要有 `asr`
  - `需修改` 后续会补对标、诊断、脚本、PDF
  - `直接放弃` 不跑 ASR
