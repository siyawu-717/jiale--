# 家乐全量明细与需修改全流程

当用户要复跑家乐千川整套流程时，使用这个 skill。

## 适用场景

- 输入两份账号近30天数据表
- 有 `信息维护表` 用于账号归属纠错
- 有 `视频ID -> oss_path` 映射表
- 需要产出全量明细、多个看板、三分类结果
- 需要对 `需修改` 素材补充对标、诊断、修改建议、两版脚本、PDF
- 用户后续再给 `pdf_url` 映射，需要回填到最终大表

## 先看流程顺序

1. 读输入表，统一字段，生成 `商品`（回味粉、小面酱、干锅酱、水煮酱；识别规则见 step1）
2. 用 `信息维护表` 更新账号归属
3. 做全量视频级汇总，输出：
   - 总体明细表
   - 看板数据表
   - 历史跑量爆款
   - 可复制
   - 需修改
   - 直接放弃
4. 对 `需修改` 素材找对标，输出：
   - 对标1 / 对标2
   - 数据定位问题
   - 具体修改建议
   - gemini优化后完整脚本建议
   - gemini优化后完整脚本建议2（微润色版）
5. 为 `需修改` 素材生成单条 PDF
6. 用户给 `pdf_url` 后，回填到 `需修改` 表和最终整表

## 固定要求

- `prompt` 不要私自改，除非用户明确要求调整
- `历史跑量爆款` 和 `可复制` 的 `asr` 要保留
- `直接放弃` 不跑 ASR
- `需修改` 的对标视频可以从：
  - `历史跑量爆款`
  - `可复制`
  里找，不受当周创建时间限制
- 对标参考的是：
  - 话术
  - 心理
  - 结构
  - 转化逻辑
  - 必要时可直接参考对标里的强句式
- 不允许把原视频主菜改掉

## 交付文件最少包含

- 一张完整大表
- `需修改` 单独分析表
- `需修改` 的 PDF 文件夹
- 如果用户给了 `pdf_url`，则回填后的最终大表
